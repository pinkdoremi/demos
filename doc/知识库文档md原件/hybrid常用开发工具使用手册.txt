####模拟数据请求，返回
有两种方式可以模拟数据。
1. 使用断点
2. 使用auto responce
注意的是目前客户端使用了请求签名，我们自己造的假数据会不被读取。解决方法：
长按客户端顶部的导航栏呼出debug工具。
Android：认证方式 0
iOS：开启混合不加密功能 开启
[fiddler使用手册进阶版](http://www.cnblogs.com/forcertain/archive/2012/11/29/2795139.html)
[无线QA处的fiddler技巧](http://172.16.2.38/qa/?p=162)
####定位国外模拟
在学会了以上两种模拟数据后，可以来模拟定位。
首先在ios和Android的debug工具中可以模拟国外的定位。
但是在ios和Android中的模拟定位的定位信息终究是假的。要真机模拟除了跑到某个经纬度城市去还有2个方法：

1. 使用越狱后的工具（fake gps），更改客户端的经纬度。但是目前这个方法是行不通的，客户端好像做了什么奇怪的操作，读不到假数据。但是如果这个方法有朝一日行得通了，那会是最有效率最方便的方式。
2. 非常不方便的方法，通过fiddler改response。

Android替换一栏：

1. 请求：`EXACT:http://loc.map.baidu.com/sdk.php`返回：`{"content":{"addr":"","bldg":"","floor":"","indoor":"1","loctp":"wf","point":{"x":"116.469953","y":"2.962493"},"radius":"75.677415","ssid":"65535"},"result":{"error":"161","time":"2015-04-07 21:06:02"}}`
2. 请求：`regex:http://api.map.baidu.com/geocoder\?location=.*&output=json`
返回：`http://api.map.baidu.com/geocoder?location=35.6894875,139.6917064&output=json`
3. 请求：`http://maps.googleapis.com/maps/api/geocode/json`返回：`http://maps.googleapis.com/maps/api/geocode/json?latlng=35.6894875,139.6917064&sensor=true&language=en`

iOS替换一览：

1. 请求：`EXACT:http://gs-loc.apple.com:443`返回：`空`
2. 安卓的2
3. 安卓的3

###[browser sync](http://www.browsersync.io/)
UI调试利器，响应式调试利器。可以监听文件的修改，刷新页面。此外还可以操作同步。
利用`npm install browser-sync`一键安装。
安装后在希望的目录
`browser-sync start --server --files="style/*.css,views/*.html"`打开服务器并且跟踪一些文件的改动。
详细的指南：
[响应式Web开发调试利器-Browser-Sync](http://www.tuicool.com/articles/JfQJvuY)
